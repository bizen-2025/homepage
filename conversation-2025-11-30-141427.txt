  ---
  Resend vs Mailgun 비교

  | 항목          | Resend          | Mailgun                 |
  |-------------|-----------------|-------------------------|
  | 무료 한도       | 3,000통/월        | 1,000통/월 (3개월간) → 이후 유료 |
  | 유료 시작가      | $20/월 (50,000통) | $35/월 (50,000통)         |
  | API 사용성     | 매우 간단           | 복잡한 편                   |
  | 설정 난이도      | 쉬움              | 도메인 인증 필수 (DNS 설정)
  |
  | React Email | 네이티브 지원         | 별도 구현 필요                |
  | 한국 발송       | 문제 없음           | 문제 없음                   |
  | 도메인 인증      | 선택              | 필수                      |

  ---
  Mailgun 장단점

  장점

  - 오래된 서비스로 안정성 높음
  - 상세한 발송 통계/로그
  - Webhook 지원 (발송 실패 알림 등)
  - 대량 발송에 적합

  단점

  - 무료 한도 적음 (1,000통 vs 3,000통)
  - 3개월 이후 유료 전환 필요
  - 도메인 DNS 설정 필수 (SPF, DKIM, MX)
  - API가 상대적으로 복잡

  코드 비교

  Resend (현재 기획)

  // lib/email.ts
  import { Resend } from 'resend';

  const resend = new Resend(process.env.RESEND_API_KEY);

  export async function sendEmail(to: string, subject: string, html: string)
   {
    return resend.emails.send({
      from: 'POLA <noreply@pola.com>',
      to,
      subject,
      html,
    });
  }

  Mailgun (교체 시)

  // lib/email.ts
  import formData from 'form-data';
  import Mailgun from 'mailgun.js';

  const mailgun = new Mailgun(formData);
  const mg = mailgun.client({
    username: 'api',
    key: process.env.MAILGUN_API_KEY!,
  });

  export async function sendEmail(to: string, subject: string, html: string)
   {
    return mg.messages.create(process.env.MAILGUN_DOMAIN!, {
      from: 'POLA <noreply@pola.com>',
      to: [to],
      subject,
      html,
    });
  }

  패키지 변경

  # Resend 제거, Mailgun 설치
  npm uninstall resend
  npm install mailgun.js form-data

  환경변수 변경

  # .env.local
  # 기존
  RESEND_API_KEY=re_XXXXXXXXXX

  # 변경 후
  MAILGUN_API_KEY=key-XXXXXXXXXX
  MAILGUN_DOMAIN=mg.pola.com

  결론: 여전히 Resend 권장

  이유:
  1. 무료 한도 3배 (3,000통 vs 1,000통)
  2. DNS 설정 더 간단 (3개 vs 5개 레코드)
  3. API 코드 절반 수준으로 간결
  4. React Email 네이티브 지원
  5. Mailgun 무료는 3개월 후 유료 전환 필수

  ---
  도메인 인증 설정 (Resend)

  도메인이 pola.com이라면:

  # DNS 레코드 추가 (3개)

  1. SPF
     Type: TXT
     Name: @
     Value: v=spf1 include:_spf.resend.com ~all

  2. DKIM
     Type: TXT
     Name: resend._domainkey
     Value: (Resend 대시보드에서 제공)

  3. Return-Path
     Type: CNAME
     Name: resend-bounces
     Value: bounces.resend.com

  설정 후 noreply@pola.com으로 발송 가능
